Ответы на контрольные вопросы:

1. Что такое репозиторий?
Репозиторий — это хранилище вашего проекта, которое содержит всю историю изменений, все версии файлов, коммиты, ветки и теги. Это как папка проекта, но с дополнительной системой контроля версий.

2. Чем отличается отслеживаемый файл от неотслеживаемого?
Отслеживаемый файл — файл, который был добавлен в Git (командой git add) и версии которого сохраняются в истории изменений

Неотслеживаемый файл — файл, который существует в папке проекта, но не добавлен в Git (не был в git add)

3. Для чего нужны git add и git commit?
git add — добавляет изменения файлов в "промежуточную область" (staging area), подготавливая их к коммиту

git commit — создает постоянный снимок подготовленных изменений с комментарием о том, что было сделано

4. Что такое ветка в Git?
Ветка — это отдельная линия разработки, которая позволяет работать над разными функциями или версиями проекта одновременно, не мешая основной кодовой базе.

5. Для чего нужен merge?
Merge (слияние) нужен для объединения изменений из разных веток. Например, когда вы завершили работу над функцией в отдельной ветке и хотите перенести эти изменения в основную ветку.

6. Чем отличаются push и pull?
git push — отправляет ваши локальные коммиты на удаленный репозиторий (например, на GitHub)

git pull — загружает изменения из удаленного репозитория и объединяет их с вашей локальной версией
